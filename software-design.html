Software Design – Animal Shelter (MongoDB)

Artifact Description

The chosen artifact is the AnimalShelter Python CRUD module, which handles database tasks for an animal rescue collection stored in MongoDB. I originally developed this project in CS 340 Module Four. The system allows users to create, retrieve, update, and delete animal records. It also includes search routines for water rescue dogs, mountain or wilderness rescue dogs, and dogs trained for disaster or individual searches.

The code is organized into a reusable class called AnimalShelter. This design allows other projects or dashboards to import and use the module without modification.

Justification for Inclusion and Enhancements

This artifact serves as evidence of my software design and engineering abilities. Specific components that demonstrate my skills include:

  • Object-Oriented Design: The AnimalShelter class encapsulates database operations with clearly defined method responsibilities.
  • Database Interaction: The system connects to MongoDB and retrieves animal records. Users can filter results by animal type, age, and breed.
  • Error Handling: The system uses try-except blocks to ensure the program does not crash unexpectedly during execution.
  • Dashboard Integration: The module supports Dash/Jupyter Dash integration, allowing real-time data visualization.

I implemented several key updates to enhance the project:

  • Replaced all print statements with Python’s logging module to improve traceability.
  • Added timestamped log records to assist with system maintenance and debugging.
  • Removed hardcoded credentials to strengthen security.
  • Added a README file explaining system design, installation, and usage instructions.

Course Outcomes

  • Software Engineering & Design: Implemented logging and improved documentation to align the project with professional coding practices.
  • Security Mindset: Removing hardcoded credentials reduces vulnerabilities and protects sensitive information.
  • Computing Solutions Design: Organized the class structure so each method has a clear responsibility, improving readability and maintainability.
  • Professional Communication: The README provides structured documentation explaining module usage and architecture.

Reflection

One of the most important improvements was enhancing maintainability through clear commenting and documentation. Descriptive comments were added to each method so that future developers — or myself — can easily understand its purpose and usage.

I also improved exception handling to clarify expected behavior under different situations and error cases. This strengthened reliability while maintaining simplicity.

A key lesson I learned is that small improvements — such as improving documentation and logging — can significantly elevate the professionalism of a project without changing its core functionality.

One challenge was ensuring comments remained accurate, concise, and meaningful without cluttering the code. Overall, these enhancements made the artifact more polished, maintainable, and reflective of my growth as a software developer.
