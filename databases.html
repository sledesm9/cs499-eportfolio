Databases – SQLite Ticket Tracker

This page includes my narrative reflection and access to the original and enhanced versions of my SQLite Ticket Tracker artifact.

Project Overview

This artifact is a support ticket tracking system built using Python and SQLite. The original version stored tickets in a basic table structure and supported simple create and read operations.

For the enhancement, I aligned the system more closely with real-world database design practices by improving data integrity, query safety, and performance. I strengthened the schema with constraints, added indexing for faster lookups, and implemented parameterized queries to mitigate SQL injection risk. I also added reporting queries so the system can generate meaningful summaries instead of simply storing raw records.

Narrative Reflection

Brief Description

This artifact was created as a ticket tracking system designed to store, update, and retrieve support tickets using SQLite. It demonstrates database schema design, structured queries, and integrating persistent storage into an application layer.

Justification for Inclusion

I selected this artifact because it demonstrates practical database implementation skills that directly translate into professional software development. The enhanced version reflects stronger database architecture, better performance optimization, and improved security awareness.

Enhancements and Skills Demonstrated

  • Improved schema design: Added constraints to protect data integrity and prevent invalid records.
  • Performance optimization: Added indexes to improve query efficiency for frequent lookups.
  • Security mindset: Used parameterized queries to mitigate SQL injection vulnerabilities.
  • Reporting and analytics: Implemented summary queries to analyze ticket trends and categories.
  • Code organization: Structured reusable database functions for maintainability.

Reflection on the Process

The most valuable lesson from this enhancement was understanding how small database design decisions can significantly impact long-term reliability and performance. By tightening schema constraints, adding indexes, and using safer query patterns, I developed a stronger appreciation for defensive programming and scalable database practices. This process strengthened my ability to think beyond functionality and consider long-term maintainability and security.

Course Outcome Alignment

  • Design and evaluate computing solutions: Improved schema design and indexing required evaluating trade-offs between simplicity and efficiency.
  • Use well-founded techniques and tools: Applied industry-standard database practices including indexing and secure query handling.
  • Develop a security mindset: Implemented parameterized queries and schema constraints to protect against vulnerabilities.
  • Professional communication: Structured documentation and code organization to clearly communicate design decisions.
